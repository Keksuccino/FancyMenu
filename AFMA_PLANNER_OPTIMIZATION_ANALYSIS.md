Good for compression, bad for planner throughput.

**Best Optimization Order**
1. Replace inner-loop archive simulation with a cheap incremental planner score, and only run full `simulateArchiveLayout()` on window winners or periodic top-K survivors.
2. Add a window-scoped candidate cache plus `(archiveState, candidate)` append-result memoization.
3. Build one shared frame-pair analysis object and reuse it across duplicate/delta/copy/multi-copy/drift logic.
4. Add strong early-outs before `detectMulti()` and `block_inter`, and stop constructing payloads until a cheap bound says they can win.
5. Replace nested common-pool parallelism with a bounded executor strategy.
6. Add heuristic skips for residual codecs, sparse layouts, and BIN_INTRA modes.

This is from static code analysis only; I did not benchmark or profile the encoder. If you want, the next best step is implementing item 1 first, because it looks like the largest speedup by far without needing to weaken the final archive packing stage.
